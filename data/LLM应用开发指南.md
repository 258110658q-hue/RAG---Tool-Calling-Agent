# LLM 应用开发实战指南

## LLM 的基本概念

大语言模型（Large Language Model，LLM）是基于 Transformer 架构、在海量文本数据上训练的深度学习模型。它们通过学习文本中的统计规律，获得了理解、生成和推理自然语言的能力。

## 主流 LLM 对比

### GPT 系列（OpenAI）
- 最早的商业化大模型
- 擅长：通用对话、代码生成、推理
- API 成熟度高，生态完善

### Claude 系列（Anthropic）
- 强调安全性和诚实性
- 擅长：长文档理解、多语言、遵循复杂指令
- Constitutional AI 训练方法

### DeepSeek 系列
- 国产开源模型
- 擅长：代码生成（Coder 版本）、数学推理
- 性价比高

### 通义千问（阿里云）
- 阿里云商业模型
- 擅长：中文场景、多模态
- 与阿里云生态深度集成

## API 调用模式

### 普通对话模式
最基础的用法：发送一条消息，LLM 返回一条回复。

```
User: "什么是 Python？"
LLM:  "Python 是一种高级编程语言..."
```

### Function Calling 模式
LLM 不直接回答，而是返回一个结构化的"工具调用指令"：
- 工具名称（如 `calculator`）
- 参数 JSON（如 `{"expression": "2+3"}`）

开发者执行工具后，将结果返回给 LLM，LLM 再综合生成最终回答。

### Streaming 模式
LLM 逐 token 返回结果，而非等全部生成完毕再返回。好处：
- 用户感知延迟更低（边生成边显示）
- 适合聊天场景

## Prompt Engineering 核心原则

### 1. 角色设定
告诉模型"你是谁"，可以显著影响回答风格和准确性。
```
你是一个经验丰富的 Python 后端工程师...
```

### 2. 结构化指令
用标题、列表、优先级编号来组织 Prompt，比纯文本段落更有效。
```
# 任务
## 步骤 1：...
## 步骤 2：...
```

### 3. 输出格式约束
明确要求输出格式（JSON、Markdown、表格），减少后续解析的难度。

### 4. 拒绝指令
显式告诉模型"在不确定时应该说不知道"，可以有效减少幻觉。

### 5. 示例（Few-shot）
提供 1-3 个问答示例，帮助模型理解预期行为。但示例越多，Token 消耗越大。

## Token 是什么

Token 是 LLM 处理文本的最小单位：
- 英文：1 个 token ≈ 0.75 个单词（"apple" = 1 token，"blueberry" = 2 tokens）
- 中文：1 个 token ≈ 0.5-1 个汉字

Token 很重要，因为：
- API 按 Token 数量计费（输入 + 输出）
- 模型有最大上下文窗口限制（如 128K token）
- Prompt 越长，每次调用的成本越高

## Agent 架构模式

### ReAct（Reasoning + Acting）
最经典的 Agent 模式：
1. Thought（思考）：LLM 分析当前状态
2. Action（行动）：调用工具
3. Observation（观察）：获取工具返回结果
4. 循环 1-3，直到得出最终答案

### Plan-and-Execute
先制定完整计划，再逐步执行。适合复杂多步骤任务。

### Multi-Agent
多个 Agent 协作，各自负责不同领域。如一个 Agent 负责代码，另一个负责文档。

## 总结

LLM 应用开发不仅仅是调用 API，更涉及 Prompt 设计、工具集成、安全控制等多个维度。选择合适的基础模型、设计有效的 System Prompt、恰当使用工具调用，是构建高质量 LLM 应用的关键。
