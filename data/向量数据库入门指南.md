# 向量数据库入门指南

## 什么是向量数据库

向量数据库是一种专门用于存储和检索高维向量的数据库系统。与传统的关系型数据库（如 MySQL）或文档数据库（如 MongoDB）不同，向量数据库的核心能力是**相似性搜索**——给定一个查询向量，快速找到与之最相似的 N 个向量。

## 为什么需要向量数据库

在大模型（LLM）应用中，我们经常需要做以下事情：

1. **语义搜索**：用户问"怎么退款"，数据库中存的是"退货政策说明"。传统关键词搜索匹配不到，但向量搜索能通过语义相似度找到。
2. **长时记忆**：LLM 的上下文窗口有限（比如 128K token），无法把所有文档都塞进去。向量数据库作为"外部记忆"，按需检索相关内容。
3. **多模态检索**：以图搜图、以文搜图等场景。

## 核心概念：Embedding

Embedding（嵌入）是将文本、图片等非结构化数据转换为数值向量的过程。

举例说明：
- 文本 "苹果" 经过 Embedding 模型后变成 → [0.12, -0.34, 0.78, ..., 0.05]（一个由 1536 个浮点数组成的向量）
- 文本 "香蕉" 经过同一个模型后变成 → [0.11, -0.31, 0.75, ..., 0.08]
- 文本 "汽车" 经过同一个模型后变成 → [-0.45, 0.67, -0.12, ..., 0.33]

你会发现 "苹果" 和 "香蕉" 的向量很接近（都是水果），而 "汽车" 的向量与它们差异很大。这就是 Embedding 的核心能力：**把语义相似性转化为向量空间中的距离**。

## 距离度量方式

在向量空间中衡量"相似"有三种常用方式：

### 余弦相似度 (Cosine Similarity)
计算两个向量之间的夹角。夹角越小（余弦值越接近 1），两个向量越相似。
- 优点：只关心方向，不关心长度，适合文本场景
- 公式：cos(θ) = (A·B) / (|A| × |B|)

### 欧几里得距离 (Euclidean Distance)
计算两个向量之间的直线距离。距离越小越相似。
- 优点：直观易懂
- 缺点：受向量长度影响大

### 点积 (Dot Product)
两个向量对应位置相乘再求和。
- 优点：计算速度快
- 注意：需要向量已归一化

## 分块策略

将长文档切分成小块（Chunk）是 RAG 系统的关键步骤。常见的分块策略：

1. **固定大小分块**：每 N 个字符切一块，最简单但可能切断句子
2. **递归分块**：先按段落分，太长的再按句子分，还长再按字符分
3. **语义分块**：用模型判断语义边界，效果最好但最慢

## 总结

向量数据库是现代 RAG 系统的核心组件。它解决了 LLM 的"知识新鲜度"和"上下文容量"两个根本性限制。选择合适的 Embedding 模型、距离度量方式和分块策略，是构建高质量 RAG 系统的关键。
